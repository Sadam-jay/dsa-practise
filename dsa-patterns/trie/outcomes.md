# Pattern 14: Trie - Learnings & Outcomes

## 📌 What is a Trie?
A **Trie** (pronounced "try"), or Prefix Tree, is a specialized tree data structure explicitly designed to store and search for **strings** extremely efficiently.
Unlike a Binary Search Tree where each node stores a full value, a Trie node stores *individual characters*. The path from the root to a specific node represents a string or prefix.
It is the absolute golden standard for problems involving Autocomplete, Prefix Matching, Spell Checking, and fast String Lookup.

## 🎯 When to Recognize It
1. The problem involves searching for strings, words, or prefixes in a very large dictionary.
2. The problem explicitly mentions "AutoComplete" or "Search Suggestions."
3. You need to find the longest common prefix among strings.
4. You are searching a 2D Matrix for multiple words simultaneously (e.g., `Word Search II`).
5. You need to maximize the XOR of two integers (special case: Bitwise Trie).

## 💡 Key Tricks
* **The TrieNode Class:** 
    1. `children = new Map()` (or an array of size 26 for strictly lowercase alphabets).
    2. `isEndOfWord = boolean` (flags when a full valid word successfully concludes).
* **Word Search Pruning:** In matrix grid DFS problems involving multiple words, build a Trie of the target words. Pass the Trie *nodes* along within the DFS! If a grid character isn't explicitly in the current Trie node's children, you beautifully and securely abort immediately (crushing time complexity limits).
* **Bitwise Trie:** A Trie doesn't just hold characters (`a-z`). It can hold bits (`0` and `1`)! When pushing 32-bit integers into a Trie to find a Maximum XOR, you logically iterate from the Highest Significant Bit down, greedily attempting to travel down the opposite bit path!
